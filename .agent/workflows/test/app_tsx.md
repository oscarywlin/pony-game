description: 測試 App.tsx 核心流程與狀態切換
---

狀態：初始為 [ ]、完成為 [x] 注意：狀態只能在測試通過後由流程更新。 測試類型：UI 流程、狀態管理、邊界條件驗證

[x] 【UI 流程】驗證登入攔截功能
範例輸入：不輸入名字，直接點擊「✨ 進入樂園！」
期待輸出：出現 GameModal 提示「先告訴我你叫什麼名字！」且留在登入頁面

[x] 【UI 流程】驗證成功登入後的跳轉
範例輸入：輸入名字「測試小馬」，點擊「✨ 進入樂園！」
期待輸出：畫面切換至主導航介面，導航列顯示使用者名稱「測試小馬」與獎勵資訊

[x] 【狀態管理】驗證科目與年級選擇同步
範例輸入：在首頁選擇「英文」科目與「3 年級」，觀察狀態
期待輸出：UI 元件顯示正確選項，且點擊「🚀 開始冒險 🚀」時發送正確參數 (可在 Console 或網路請求觀察)

[x] 【UI 流程】驗證商店介面切換
範例輸入：在主頁點擊「🏪 造型商店」
期待輸出：畫面顯示商店介面內容，且可透過關閉按鈕回到首頁

[x] 【邊界條件】驗證 API 失敗時的 Fallback
範例輸入：模擬 GAS API 回傳錯誤（或斷網環境）嘗試登入
期待輸出：系統應進入 Local 模式開發（Mock 登入），允許使用者進入主頁面
